# 股票系统 - 运维手册

## 一、数据库维护

### 1.1 分区自动创建

**初始化时自动创建**：
- 当前年份和下一年的日K线分区（共 2 年）
- 当前月、下月、下下月的分钟K线分区（共 3 个月）

**定时任务**：使用 Django Celery Beat 定时执行分区创建任务（需配置 Celery）

### 1.2 分区状态检查（建议每月执行一次）

```sql
-- 查看已创建的日K线分区
SELECT tablename FROM pg_tables WHERE tablename LIKE 'stock_daily_kline_%' ORDER BY tablename;

-- 查看已创建的分钟K线分区
SELECT tablename FROM pg_tables WHERE tablename LIKE 'stock_minute_kline_%' ORDER BY tablename;

-- 查看定时任务状态
SELECT * FROM cron.job;

-- 查看定时任务运行日志（最近10条）
SELECT * FROM cron.job_run_details ORDER BY start_time DESC LIMIT 10;
```

### 1.3 分区创建失败时的手动补建

```sql
-- 手动创建单个年份分区
SELECT create_daily_partition_if_not_exists('2027-01-01');

-- 手动创建单个月份分区
SELECT create_minute_partition_if_not_exists('stock_minute_kline', '2026-06-01');

-- 批量创建多个年份分区（如未来5年）
SELECT create_daily_partitions('2027-01-01', 5);

-- 批量创建多个月份分区（如未来6个月）
SELECT create_minute_partitions('stock_minute_kline', '2026-06-01', 6);
```

### 1.4 数据清理策略

- **分钟K线**：建议保留最近 30 天，超过 30 天的数据归档到冷存储
- **日K线**：长期保留（分区表查询效率高）

## 二、系统配置管理

### 2.1 查看当前配置

```sql
-- 查看所有系统配置
SELECT config_key, config_value, description FROM sys_config ORDER BY config_key;

-- 查看特定配置
SELECT config_value FROM sys_config WHERE config_key = 'partition_settings';
```

### 2.2 修改配置

```sql
-- 修改分区创建配置（日K线提前3年，分钟K线提前6个月）
UPDATE sys_config 
SET config_value = '{"daily_years_ahead": 3, "minute_months_ahead": 6}'
WHERE config_key = 'partition_settings';

-- 添加新的策略类型
UPDATE sys_config
SET config_value = '["MA_CROSS", "MACD", "RSI", "KDJ", "BOLL", "CUSTOM", "NEW_STRATEGY"]'
WHERE config_key = 'strategy_types';
```

### 2.3 通过 Django API 管理配置

```bash
# 查看所有配置
GET /api/sys-config/

# 查看特定配置
GET /api/sys-config/get_by_key/?key=partition_settings

# 修改配置（需要管理员权限）
PUT /api/sys-config/{id}/
{
    "config_key": "partition_settings",
    "config_value": {"daily_years_ahead": 3, "minute_months_ahead": 6}
}
```

## 六、常见问题排查

### 6.1 分区创建失败

**现象**：写入数据时报错 `no partition of relation "stock_daily_kline" found for row`

**原因**：pg_cron 任务未执行或执行失败

**解决**：
1. 检查 pg_cron 是否启用：`SELECT * FROM cron.job;`
2. 手动创建分区：`SELECT create_daily_partition_if_not_exists('2027-01-01');`
3. 检查 PostgreSQL 日志：`docker-compose logs postgres`

### 6.2 枚举值不匹配

**现象**：插入数据时报错 `invalid input value for enum user_role`

**原因**：数据库枚举类型与 Django 模型 choices 不一致

**解决**：
1. 查看数据库枚举值：`SELECT enumlabel FROM pg_enum WHERE enumtypid = 'user_role'::regtype;`
2. 添加缺失的枚举值：`ALTER TYPE user_role ADD VALUE 'NEW_ROLE';`
3. 同步更新 Django 模型

### 6.3 pg_cron 未执行

**现象**：定时任务状态显示失败或未执行

**解决**：
1. 检查 pg_cron 扩展是否安装：`\dx pg_cron`
2. 检查 PostgreSQL 配置：`SHOW cron.database_name;`
3. 重启 PostgreSQL 服务
